## 웹 애플리케이션(Web Application)
`웹 애플리케이션` : 웹 브라우저 상에서 HTTP프로토콜 및 HTML 문서를 바탕으로 수행되는 애플리케이션<br>

<br>

### 웹 애플리케이션 구조
![alt text](../image/웹구조.png)
- **Client Tier**<br>
웹 브라우저가 해당하는 층으로 사용자가 실제로 접하는 프로그램<br>
- **Server Tier**<br>
동적 웹 컨텐츠 기술(파이썬, JSP, CGI)<br>
Client Tier와 Data Tier을 연결해 주는 소프트웨어 또는 하드웨어<br>
애플리케이션 서버는 **Presentation, Business Logic, Data Access**로 나뉨
- **Data Tier**
하부단의 저장매체로 파일 서버나 데이터 베이스 서버 등 하드웨어 장비<br>
<br>

### 요청과 흐름
1. 웹 브라우저가 웹 서버에게 특정 페이지를 요청
2. 해당 웹 서버는 웹 브라우저의 요청을 받아서 요청된 페이지의 로직 및 DB와의 연동을 위해 웹 애플리케이션 서버에 처리 요청
3. 웹 애플리케이션 서버는 DB와 연동 필요하면 DB와의 데이터 처리 수행
4. 로직 및 DB 작업의 처리 결과 웹 서버에 리턴
5. 웹 서브는 결과를 다시 웹 브라우저에게 응답<br>
<br>

### 웹 애플리케이션 관련 기술
![alt text](../image/스크립트.png)
![alt text](../image/클라이언트비교.png)
#### 1) Client Tier 기술
- **HTML(Hyper Text Markup Language)** : 여러 태그로 구성되어 있으며 만들어 갈 수 있음, Hyper Text의 원리 이용해 여러 문서 링크래 다양한 정보 쉽게 검색 가능<br>
- **HTML5**: 문서 중심에서 그림, 동영상, 음악까지, 모바일 환경에서 운영체제를 가리지 않고 모두 호환<br>
- **JavaScript** : 사용자와 페이지 간의 동적 상호작용이 가능하게 하기 위해 사용하는 스크립트 언어<br>
- **RIA(Rich Internet Application)**: 기존의 웹 애플리케이션의 편리한 배포, 관리, 저렴한 유지 비용 등의 장점과 Desktop 수준의 사용자 편의성, 데이터 통신 등의 장점을 합침<br>
필요한 부분만 로딩 해 페이지 부하 적고 빠르며, 복자한 디자인에 대한 렌더링 속도 저하 문제 극복<br>
- **Ajax(Asynchronous JavaScript and XML)** : 대화식 웹 애플리케이션을 위해 아래와 같은 조합 이용
    - HTML, CSS
    - 동적인 화면 출력 및 상호작용 위한 DOM,자바스크립트, XML,XSLT
    - XMLHttpRequest
    - 구글맵, 자동단어추천 등 실시간 정보 전달
#### 2) Server Tier 기술
- ASP(Active Server Page)
- PHP(Professional Hypertext Preprocessor)
- JSP(Java Server Page)
- Servelet
- Node.js
